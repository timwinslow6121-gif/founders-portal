SSH into the VPS. Run all commands as the deploying user (sudo available).

Step 1 — Back up SQLite database FIRST before anything else:
```bash
cd /home/founders/founders-portal   # adjust path if different
cp instance/founders_portal.db instance/founders_portal.db.backup
ls -lh instance/founders_portal.db instance/founders_portal.db.backup
```
Confirm the backup file exists and matches size before proceeding.

Step 2 — Check Ubuntu version:
```bash
lsb_release -cs   # returns e.g. "jammy" (22.04) or "focal" (20.04)
```

Step 3 — Install PostgreSQL 16 from PGDG:
```bash
sudo sh -c 'echo "deb https://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
sudo apt update
sudo apt install -y postgresql-16
sudo systemctl enable postgresql
sudo systemctl start postgresql
sudo systemctl status postgresql   # must show "active (running)"
```

Step 4 — Create database and user:
```bash
sudo -u postgres psql <<'SQL'
CREATE DATABASE founders_portal;
CREATE USER founders_user WITH ENCRYPTED PASSWORD 'CHOOSE_STRONG_PASSWORD_HERE';
GRANT ALL PRIVILEGES ON DATABASE founders_portal TO founders_user;
\c founders_portal
GRANT ALL ON SCHEMA public TO founders_user;
SQL
```
Replace CHOOSE_STRONG_PASSWORD_HERE with a strong password. Note it — you'll need it for DATABASE_URL.

Step 5 — Install pgloader:
```bash
sudo apt install -y pgloader
pgloader --version   # must print version >= 3.6.x
```
If version is below 3.6.0, use the Docker image instead: `docker pull dimitri/pgloader:latest`

Step 6 — Pull the latest code from git (to get migrations/ directory from Plans 01+02):
```bash
cd /home/founders/founders-portal
git pull origin main
ls migrations/versions/   # must show 3 files: 001, 002, 003
```
