<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Add 2GB swap file and update Gunicorn threading config</name>
  <what-built>N/A — this task IS the human action on the VPS.</what-built>
  <how-to-verify>
    On the VPS. All commands via SSH.

    Step 1 — Add 2GB swap file:
    ```bash
    # Check if swap already exists
    swapon --show
    free -h

    # If no swap, create it:
    sudo fallocate -l 2G /swapfile
    # If fallocate fails on this VPS:
    # sudo dd if=/dev/zero of=/swapfile bs=1M count=2048

    sudo chmod 600 /swapfile
    sudo mkswap /swapfile
    sudo swapon /swapfile

    # Make persistent across reboots
    echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

    # Verify
    swapon --show
    free -h   # should show ~2G swap
    ```

    Step 2 — Find and update Gunicorn systemd unit:
    ```bash
    systemctl show founders-portal --property=FragmentPath
    # Note the path returned, e.g. /etc/systemd/system/founders-portal.service
    sudo nano /etc/systemd/system/founders-portal.service   # adjust path as needed
    ```

    In the [Service] section, update the ExecStart line to:
    ```
    ExecStart=/home/founders/venv/bin/gunicorn \
        --workers 2 \
        --threads 4 \
        --worker-class gthread \
        --timeout 120 \
        --bind 127.0.0.1:8000 \
        wsgi:app
    ```
    Adjust the venv path to match the actual venv location on this VPS.

    Step 3 — Reload systemd and restart:
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl restart founders-portal
    sudo systemctl status founders-portal   # must show "active (running)"
    ```

    Verification checks:
    - `swapon --show` shows /swapfile 2G
    - `free -h` shows ~2G swap
    - `grep /swapfile /etc/fstab` returns the fstab entry (persistent)
    - `systemctl cat founders-portal | grep gthread` shows the gthread worker class
    - `sudo systemctl status founders-portal` shows "active (running)"
  </how-to-verify>
  <resume-signal>Type "task2-done" after swap file is active, fstab updated, Gunicorn config updated, and app restarted successfully.</resume-signal>
</task>
